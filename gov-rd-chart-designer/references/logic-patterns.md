# R&D Proposal Logic Patterns

Use these patterns to strengthen government science-and-technology R&D proposal figures.

## Application Logic Chain

Use when the figure must prove project necessity and feasibility:

`政策需求/行业痛点 -> 科学问题/技术瓶颈 -> 研发思路 -> 关键技术 -> 研发任务 -> 成果产出 -> 应用示范 -> 绩效指标`

Design notes:

- Make the bottleneck explicit.
- Show which technology solves which problem.
- Connect tasks to measurable results.
- End with indicators, deliverables, demonstration, or application value.

## Technical Principle Chain

Use when the figure must show how the technology works:

`研究对象/数据/材料 -> 机理/模型/算法/工艺 -> 方法路径 -> 系统集成 -> 验证评价 -> 工程应用`

Design notes:

- Start from the actual research object, not a slogan.
- Name the mechanism, model, algorithm, process, or technical method.
- Show transformation from input to capability.
- Include verification or evaluation when the proposal claims feasibility.

## Key Technology Expression

Prefer concrete terms:

- `多源数据融合`
- `机理模型构建`
- `关键参数识别`
- `智能算法优化`
- `工艺过程调控`
- `系统集成验证`
- `示范场景应用`
- `指标量化评价`

Avoid vague terms unless supported by concrete modules:

- `全面赋能`
- `智慧提升`
- `协同推进`
- `高质量发展`
- `全方位支撑`

## Common Figure Arguments

Use one argument per figure:

- Necessity: `痛点 -> 瓶颈 -> 需突破的科学问题`.
- Feasibility: `已有基础 -> 方法路径 -> 验证条件 -> 预期指标`.
- Innovation: `传统方法局限 -> 本项目新机制/新模型/新工艺 -> 性能提升`.
- Route: `任务递进 -> 技术集成 -> 验证应用`.
- Assessment: `任务 -> 成果 -> 指标 -> 验收证据`.

## Matrix Cell Writing

Use compact, parallel wording for document matrices:

- Keep each cell to 1-3 short lines.
- Use noun phrases or verb-object phrases instead of full sentences.
- Keep rows structurally parallel so reviewers can scan across columns.
- Add concrete evidence words such as `数据集`, `评价报告`, `质控记录`, `验证日志`, `样机/原型`, or `测试报告` when the result column would otherwise look empty.
- Do not pad cells with slogans. If the table looks sparse, improve the technical content or reduce the visual row height.

## Language Tone

Use formal, precise, proposal-friendly Chinese:

- `围绕...技术瓶颈`
- `构建...模型/方法/体系`
- `形成...关键技术`
- `实现...能力提升`
- `支撑...示范应用`
- `满足...考核指标`

Avoid inflated claims without evidence. If a claim lacks technical support, revise the figure to show the missing mechanism, method, or validation link.
